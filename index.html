线程同步

发表于 2018-11-28 | 分类于 Linux

线程：
创建线程之后，地址空间没有变化
进程退出成了线程-主线程
创建出的子线程和主线程共用地址空间
主线程和子线程有各自独立的pcb
-子线程的pcb是从主线程拷贝来的
主线程与子线程共享：
.txt、.bss、.data、堆、动态库加载区、环境变量区、命令行参数
不共享：
栈被各个线程均分

线程相关：
自行安装线程man page，命令：
sudo apt-get install manpages-posix-dev
察看指定线程的LWP号：
线程号和线程ID是有区别的
线程号是给内核看的
查看方式:找到程序的进程ID:ps -Lf pid
多线程和多进程区别：
多进程：
始终共享的资源
代码、文件描述符、mmap
线程共享：堆、全局变量
线程节省资源

编译：gcc xxx.c -lpthread
创建线程函数：pthread_create()
输出自己的线程号：pthread_self()
打印错误码的错误信息strerror()
单个线程退出pthread_exit()
阻塞等待线程退出，获取线程退出状态
回收子线程pthread_join()
线程分离pthread_detach()
调用该函数之后不需要pthread_join
子线程会自动回收自己的pcb
杀死(取消)线程pthread_cancel()
pthread_testcancel()
判断线程ID是否相等pthread_equal()

线程分离：
pthread_attr_init()
pthread_attr_setdetachstate()
PTHREAD_CREATE_DETACHED分离
PTHREAD_CREATE_JOINABLE非分离
pthread_attr_destroy()
数据混乱原因：操作了共享资源、cpu调度问题
解决：线程同步

锁：
如果锁是锁着的：
线程阻塞，阻塞在这把锁上
如果锁是打开的：
线程访问共享资源，然后锁上锁
本来多线程访问共享资源时候，可以并行
通过加锁机制并行->串行

互斥锁:串行
创建互斥锁：pthread_mutex_t mutex;
初始化互斥锁pthread_mutex_init()
销毁互斥锁pthread_mutex_destroy()
加锁pthread_mutex_lock()阻塞
尝试加锁pthread_mutex_trylock()不阻塞
解锁pthread_mutex_unlock()
互斥锁同步线程：所有线程都需要加锁
原子操作：
cpu处理一个指令，线程/进程在处理完这个指令之前是不会失去cpu的

读写锁：
pthread_rwlock_t lock;
初始化读写锁pthread_rwlock_init()
pthread_rwlock_destroy()
pthread_rwlock_rdlock()
pthread_rwlock_tryrdlock()
pthread_rwlock_wrlock()
pthread_rwlock_trywrlock()
pthread_rwlock_unlock()
读写锁的类型：
读锁-对内存做读操作
写锁-对内存做写操作

条件变量：
不是锁，但是条件变量能够阻塞线程
使用条件变量+互斥量
主要函数：
初始化一个条件变量pthread_cond_init()
销毁一个条件变量pthread_cond_destroy()
唤醒至少一个阻塞在条件变量上的线程pthread_cond_signal()
阻塞等待一个条件变量pthread_cond_wait()
限时等待一个条件变量pthread_cond_timedwait()

信号量：并行
头文件：semaphore.h
信号量类型：sem_t sem加强版的互斥锁
初始化信号量：sem_init()
0-线程同步
1-进程同步
value-最多有几个线程操作共享数据
销毁信号量：sem_destroy()
加锁：sem_wait()
尝试加锁：sem_trywait()
限时尝试加锁：sem_timedwait()
解锁：sem_post()

进程的五种状态：
初始态、就绪态、运行态、挂起态、终止态

fork()创建子进程
父进程成功，fork()返回子进程的PID
子进程成功，fork()返回0
getpid() 获取进程ID
getppid()获取父进程ID
父子间进程不能通过全局变量共享内存，线程是可以的
exec函数族：需要注意的是exec并不是1个函数, 其实它只是一组函数的统称, 它包括下面6个函数
execl、 execlp、 execle、execv、 execvp、 execve
exec函数族重新替换.txt中的代码，执行exec中的代码

孤儿进程
爹先死，孩子还活着，孩子叫孤儿进程
孤儿被init进程领养，init进程变成孤儿进程的父亲
为了释放子进程占用的系统资源
进程结束之后，能够释放用户区空间
释放不了pcb，必须由父进程释放

僵尸进程
孩子死了，爹还活着，爹不去释放子进程的pcb，孩子就变成了僵尸进程
是一个死了的进程
回收进程：
wait() 阻塞函数
waitpid()
WIFEXITED(status);为非0进程正常结束
WEXITSTATUS(status) status为子进程退出状态
WIFSIGNALED(status);为非0进程异常终止
WTERMSIG(status)

进程间通信IPC
常用几种方式：
管道-简单
信号-系统开销小
信号量
消息队列
共享映射区-有无血缘关系的进程间通信都可以
本地套接字-稳定

匿名管道：pipe
本质：内核缓冲区、伪文件不占用磁盘空间
特点：
两部分：读端，写端对应两个文件描述符
数据写端流入，读端流出
操作管道的进程被销毁之后，管道自动被释放了
管道默认是阻塞的
使用场景：只能血缘关系进程间通信
管道的原理：
内部实现方式：队列 环形队列 fifo
缓冲区：默认4K
管道的局限性：
队列：数据只能读取一次，不能重复读取
半双工
设置读端为非阻塞：
int flags = fcntl(fd[0],F_GETFL);
flags |= O_NONBLOCK;
fcntl(fd[0],F_SETFL,flags);

有名管道：fifo
特点：在磁盘上有这样一个文件p
伪文件，在磁盘大小永远为0
在内核中有一个对应的缓冲区
半双工的通信方式
使用场景：无血缘关系进程间通信
创建方式：mkfifo 管道名

内存映射:mmap
mmap()建立内存映射区
munmap()释放内存映射区
匿名内存映射区(父子间进程通信)

信号：signal
特点：简单，携带的信息量少，使用在某个特定的场景中
信号的状态：
产生：键盘、命令kill、系统函数、定时器、段错误，除0错误
未决状态：没有被处理
递达：信号被处理了

信号处理：忽略、捕捉、执行默认动作
概念：阻塞信号集、未决信号集
相关函数：
kill() 发送信号给指定进程
raise()自己给自己发信号
abort()给自己发送给异常终止信号
闹钟(定时器)：
alarm()设置定时器(每个进程只有一个定时器)
settimer()定时器，并实现周期性定时
自定义信号集
sigemptyset()将set集合置空
sigfillset()将所有信号加入set集合
sigaddset()将signo信号加入到set集合
sigmember()判断信号是否存在
sigpending() 读取当前进程的未决信号集
sigprocmask()自定义信号集设置给内核阻塞信号集
signal()注册捕捉函数
sigaction()捕捉信号

守护进程：
一般步骤：
先fork,夫进程死，子进程执行创建会话操作
setsid()设置会话组
改变当前工作目录chdir
重设文件掩码umask(0)
关闭文件描述符0,1,2
执行核心工作

进程间同步：信号量、互斥锁、文件锁

内核最后阶段启动的三个进程

发表于 2018-11-26 | 分类于 Linux

在内核启动的最后阶段启动了三个进程
进程0：进程0其实就是刚才讲过的idle进程，叫空闲进程，也就是死循环。

进程1：kernel_init函数就是进程1，这个进程被称为init进程。

进程2：kthreadd函数就是进程2，这个进程是linux内核的守护进程。这个进程是用来保证linux内核自己本身能正常工作的。

在嵌入式操作系统中一般会指定/linuxrc为init进程。
